# v0.1.7

## Summary
- 对齐 Autonomous Agent 的 auto/manual 双投递模式设计，正式把 `REVIEW` 审批阶段纳入核心规范、AIEF 计划与 GUI 设计真源。
- 明确 `delivery_mode`、`batch_review`、`run.agent.getPendingReview`、`run.agent.submitReview` 与 `agent.review.*` 事件语义，为后续实现提供统一文档基线。

## Highlights
- 更新 `openspec/changes/autonomous-agent-delivery-loop/` 与 `openspec/specs/pi-proof-forge-core.md`，统一 10 状态机、`ReviewStage`、`ReviewCandidate`、`ReviewDecision` 定义。
- 更新 `AIEF/workflow/phases/submission.md`、`AIEF/context/tech/GUI_ARCHITECTURE.md`、`AIEF/docs/plans/backend-ui-bridge-plan.md`、`AIEF/docs/plans/autonomous-agent-delivery-loop-v2.md`，把手动审批路径、批量审批模式与 bridge/RPC 需求写实。
- 更新 `ui/design/DESIGN.md`，补齐 Agent Run 的 REVIEW 审批面板、Policy 页面中的 `delivery_mode` / `batch_review` 字段，以及 REVIEW 相关 RPC 合约。

## Breaking Changes
- Autonomous Agent 的设计规范从 9 状态流更新为 10 状态流；后续任何实现、评审和验收都应以 `GATE -> REVIEW -> DELIVER` 的新流程为准。

## Migration Guide
- 实现 `run_agent`、sidecar RPC、Agent Run 页面时，统一遵循 `delivery_mode=auto|manual` 与 `batch_review=true|false` 的组合语义。
- REVIEW 相关事件统一使用 `agent.review.pending` 与 `agent.review.resolved`，避免再引入 `review.*` 的并行命名。
- 文档与设计评审时，以 `openspec/changes/autonomous-agent-delivery-loop/design.md`、`openspec/specs/pi-proof-forge-core.md` 与 `ui/design/DESIGN.md` 为主参考。
