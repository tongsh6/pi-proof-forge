# v0.1.3

## Summary
- 将 release notes 纳入标准发布流程，发布不再默认依赖自动生成 notes。
- 发布命令、skills、技术文档和工具说明同步对齐到同一口径。

## Highlights
- `tools/run_github_publish.py` 新增 `--release-notes-file`，默认优先读取 `release-notes/<version>.md`。
- 新增显式回退开关 `--allow-generate-notes`，仅在明确允许时才使用 GitHub auto-generated notes。
- `/publish-release` 命令在 opencode/claude/cursor 三端统一为四参数：
  - `<feature> <release> <version> <notes-file>`

## Breaking Changes
- 发布命令行为变更：未提供 release notes 文件且未显式开启 `--allow-generate-notes` 时，发布将被阻断。

## Migration Guide
- 旧用法：
  - `python3 tools/run_github_publish.py --feature <feature> --release <release> --version <version>`
- 新用法（推荐）：
  - `python3 tools/run_github_publish.py --feature <feature> --release <release> --version <version> --release-notes-file release-notes/<version>.md`

## Verification
- `python3 tools/run_github_publish.py --help`
- `python3 tools/run_github_publish.py --feature demo --release v0.1.3 --version v0.1.3 --release-notes-file release-notes/v0.1.3.md --dry-run`
- `python3 tools/check_aief_l3.py --root . --base-dir AIEF`
