# v0.1.9

## Summary
- 完成 `autonomous-agent-delivery-loop` 变更的 A-E 全阶段落地：统一核心引擎地基、引擎注册表、编排与通道、CLI 薄入口、静态约束与测试收口。

## Highlights
- **Domain + Config 基础层**：新增 `invariants/events/run_state`、`errors`、`config loader/validator/composer`、结构化日志与 run store。
- **Engine v2**：新增 `tools/engines/` 全家桶（evidence/matching/generation/evaluation/discovery + registry），并完成策略注册接线。
- **Orchestration + Delivery**：新增 `stage/pipeline/state_machine/gate/review/agent_loop`，支持 strict/simulate、N-pass、manual/auto review、`max_deliveries` 安全停止。
- **Channels**：新增 `liepin` 与 `email` 通道，以及可重试错误下的降级策略。
- **CLI 收口**：新增 `tools/cli/entrypoints.py` 与命令分发；旧入口脚本兼容转调新命令。
- **去重与约束**：将重复 LLM/YAML 逻辑迁移到 `tools/infra/llm/client.py` 与 `tools/infra/persistence/yaml_io.py`，新增 `tools/check_v2_constraints.py` 静态校验。
- **文档与规范**：更新 GUI 配置承载边界（A10a）与 v2 CLI 文档；`openspec/changes/autonomous-agent-delivery-loop/tasks.md` 全项完成（unchecked=0）。

## Breaking Changes
- None. 旧入口兼容保留，默认转调新 CLI 命令。

## Migration Guide
- 推荐新入口：
  - `python3 -m tools.cli.entrypoints extract|match|generate|evaluate|pipeline|agent ...`
- 旧入口仍可用：
  - `python3 tools/run_pipeline.py ...`
  - `python3 tools/run_agent.py ...`
- 如需强制走旧实现（兼容对比/排障）：
  - 设置环境变量 `PPF_FORCE_LEGACY_MAIN=1`

## Verification
- `python3 -m pytest tests/unit/domain tests/unit/infra tests/domain tests/unit/pipeline tests/unit/matching -q`
- `python3 tools/check_aief_l3.py --root . --base-dir AIEF`
- `python3 tools/check_v2_constraints.py --root .`
- `python3 tools/run_agent.py --policy /tmp/ppf_policy.yaml --dry-run --run-id run-final-check --output-dir /tmp/ppf_agent_runs_final`
- `python3 -m tools.cli.entrypoints pipeline --raw tools/sample_raw.txt --job-profile job_profiles/jp-2026-001.yaml --run-id run-final-pipeline-2`
