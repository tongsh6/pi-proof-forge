# GUI 原型评审说明

原型文件：`ui/prototype/index.html`

## 评审目标
- 页面是否覆盖核心闭环：输入 -> 执行 -> 结果预览。
- 交互是否符合你的操作路径：切换 LLM、严格模式、查看 A/B 与评分。

## 核心交互
1. 输入原始语料来源：支持多个目录与多个文件上传。
2. 岗位画像可导入（yaml/json）并在页面上编辑。
3. 勾选 `使用 LLM 增强` / `严格 LLM`，点击 `执行 Pipeline`。
4. 右侧查看纯文本流式日志（Reasoning Trace），包含系统状态、执行动作、思考过程与结果。
5. 切换 Tabs 查看 `Match / Resume A / Resume B / Scorecard`。
6. 点击右上角 `中文/EN` 进行语言切换（严格隔离，无中英混排）。

## 重点确认项
- 信息层级是否清楚。
- 结果预览是否满足“快速判断是否可投”。
- 哪些字段必须上屏（例如 gaps、gap tasks、version_id、run 命令）。
- 流式日志（思考链风格）是否比进度条更能体现 AI 执行过程。
- 岗位画像编辑区是否需要拆成“基础字段 + 高级字段”。
