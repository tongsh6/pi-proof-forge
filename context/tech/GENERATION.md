# 简历生成（Generation）

## 目标
- 模板优先拼装，确保事实不被改写。
- 受控改写只做措辞与关键词对齐，不新增事实。
- 输出可版本化（resume_version_id）。

## 固定结构（1 页优先）
1) 顶部 10 秒区：定位 + 核心标签 + 技术栈
2) 核心亮点 3 条：结果 + 场景 + 动作
3) 工作经历：每段 4–6 bullets（动词 + 影响面 + 方法/约束 + 结果）
4) 代表项目 3 个：挑战-动作-结果
5) 开源/工具/内容（可选）

## 生成流程（强约束）
1. 召回证据卡
2. 排序并选 Top N
3. 模板拼装（事实字段锁定）
4. LLM 受控改写（仅措辞/关键词）
5. 生成版本号 `resume_version_id`

## 输出版本
- 版本 A：偏架构/技术深度
- 版本 B：偏交付治理/Owner

## 事实保真规则
- 不允许新增指标、范围、职责或角色级别。
- 所有关键表达必须可追溯到 evidence cards 字段。

## 可配置项（建议）
- 模板样式与版式
- 关键词对齐策略
- 受控改写的允许变更清单

## LLM 辅助模式
- CLI：`tools/run_generation.py --use-llm`
- Prompt：`tools/prompts/generation.md`
- 严格模式：`--use-llm --require-llm`（LLM 不可用即失败）

## 当前实现状态
- 默认使用模板化规则拼装（A/B 两版可复现）。
- `--use-llm` 打开时优先使用 LLM 生成；失败回退模板输出。
