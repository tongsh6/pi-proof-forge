# 开源发布需要先补 README 与 LICENSE

**日期**：2026-03-04
**场景**：项目已推送到 GitHub 公共仓库，但根目录缺少 README 与 LICENSE。

## 问题

- 无根目录 README 时，首次访问者无法快速理解项目定位、运行方式与目录结构
- 无 LICENSE 时，开源仓库的使用、分发与修改权限边界不清晰

## 处理

- 新增根目录 `README.md`，补齐项目价值、核心流程、快速开始、目录说明与治理入口
- 新增 `LICENSE`（MIT），明确开源授权边界
- 在 README 中显式标注“submission 自动投递仍处于规划状态”，避免功能误解

## 经验

- “代码先公开，文档后补”会显著降低仓库可用性，应在开源首发时同步提供 README + LICENSE
- 对未实现能力要用明确状态词（如 planning）标注，减少外部使用者预期偏差
