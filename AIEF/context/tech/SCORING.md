# 匹配与评分（Scoring）

## 目标
- 可解释评分，输出分项得分与理由。
- 结果稳定可复现，支持版本化。
- 产出缺口清单与补证据任务。

## 评分维度（默认）
- **关键词覆盖（K）**：技术栈词、职责词、领域词
- **业务域相似度（D）**：会员/营销/门店/电商等
- **级别信号（S）**：Owner、决策、跨团队推进、带人
- **量化强度（Q）**：指标数量与质量（含可验证性）
- **证据强度（E）**：artifacts 完整度与可信度
- **新近性（R）**：近 12–18 个月加权

## 输入
- Job Profile
- Evidence Cards
- 评分权重配置（可选）
- LLM（可选）：用于生成可解释理由

## 输出
- 总分 + 分项得分（可解释）
- Top N 证据卡（含入选理由）
- 缺口清单（缺关键词/缺能力证据/缺指标）
- 补证据任务（例如补压测结果、补监控截图、补复盘链接）

## LLM 辅助模式
- CLI：`tools/run_matching_scoring.py --use-llm`
- Prompt：`tools/prompts/matching-scoring.md`
- 严格模式：`--use-llm --require-llm`（LLM 不可用即失败）

## 当前实现状态
- 默认使用规则打分（可复现）：K 25 / D 15 / S 15 / Q 20 / E 15 / R 10。
- `--use-llm` 打开时优先使用 LLM 输出；若失败自动回退规则打分。

## 规则约束
- 缺少 results 或 artifacts 的证据卡不得进入候选池。
- 所有分项得分必须可解释并可追溯到字段。

## 可配置项（建议）
- 维度权重
- 近半年/近一年加权策略
- Top N 候选数量
