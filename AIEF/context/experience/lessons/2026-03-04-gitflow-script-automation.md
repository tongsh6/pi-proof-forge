# GitFlow 要落到可执行脚本

**日期**：2026-03-04
**场景**：用户要求将 GitFlow 固化为可执行的跨平台发布脚本，并明确分支保留策略。

## 问题

仅有 GitFlow 文档时，团队执行仍依赖人工记忆，容易出现：
- 分支流转顺序不一致
- 合并后忘记删除 `feature/*`
- 发布后误删 `release/*`

## 处理

- 新增脚本：`tools/run_gitflow_release.py`
- 固化主流程：`main -> feature -> develop -> release -> main`
- 固化策略：`feature/*` 合并后删除，`release/*` 合并后保留
- 脚本支持 `--dry-run`，便于发布前预演

## 经验

- 发布流程文档化后，应立即补对应自动化脚本，否则规范执行一致性不高
- 分支保留策略应参数化且设置合理默认值，避免“策略在文档、行为在习惯”
