# 自动投递阶段先文档化再实现

**日期**：2026-03-04
**场景**：用户明确期望完整闭环（语料加工 -> JD 匹配 -> 简历生成 -> 自动投递），当前仓库缺少投递阶段定义。

## 问题

现有 workflow 只覆盖到 `evaluation`，没有 `submission` 阶段，导致“可生成简历”与“可完成实际投递”之间存在断层。

## 处理

- 新增 `workflow/phases/submission.md`，明确自动投递阶段的目标、前置条件、步骤、产出、CLI 约定与验收标准
- 更新 `workflow/INDEX.md`，把自动投递加入核心流程与脚本映射
- 在 `tools/README.md` 增加“自动投递（规划中）”命令约定，避免实现前的使用歧义

## 经验

- 在涉及外部平台自动化（Playwright）的需求中，先文档化“输入/输出/日志/重试”边界，可显著降低后续实现返工
- 对尚未实现的命令，必须在文档中显式标注“规划中”，避免被误认为可直接执行
